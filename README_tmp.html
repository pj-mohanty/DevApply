DevApply

Track job applications, log interview questions, and analyze your tech interview journey.

DevApply is a full-stack web application designed to help software engineers track job applications and document real interview experiences — including coding questions, system design prompts, and behavioral notes.

Built by engineers, for engineers.

🚀 Features

  • 📝 Job Application Tracker – Manage companies, roles, status, links, and stages.
  • 💬 Interview Journal – Record real coding, system design, and behavioral questions.
  • 📅 (Optional) Interview Scheduling – Keep track of upcoming and past interviews.
  • 📄 (Optional) Resume Version Tracking – Upload and log which version was submitted to each job.
  • 📊 (Optional) Analytics Dashboard – Visualize trends in your interview history.
  • 🤖 (Optional) AI Enhancements – Resume/job match, question tagging, summarization.

🛠️ Tech Stack

  • Frontend: React, Tailwind CSS
  • Backend: Node.js, Express.js
  • Database & Auth: Firebase / Firestore
  • Deployment: Vercel or Render
  • Version Control: GitHub

📂 Getting Started

Coming soon: setup instructions and environment config.

  1. Clone the repo
git clone https://github.com/dev-apply/devapply.git
cd devapply
  1. Install Dependencies Install the required packages for both the frontend and backend:
# Frontend
cd client
npm install

# Backend
cd ../server
npm install

  1. Configure Firebase To connect to Firestore:

    1.Go to Firebase Console and create a project.

    2.Enable Cloud Firestore under Build > Firestore Database.

    3.Navigate to Project Settings > Service Accounts, click Generate new private key, and download the JSON file.

    4.Save the file as:

    server/firebaseServiceAccountKey.json

  2. Set Up OpenAI API Key
    1.In the server/ directiory, create a .env file

    touch server/.env

    2.Add the following line to .env file(replace your-api-key-here with your actual API key):

    OPENAI_API_KEY=your-api-key-here

    ⚠️ Make sure .env is listed in .gitignore to avoid committing it to Git.

  3. Seed Mock Data (Optional) To populate the database with sample job applications and interview entries:

cd server
node scripts/seedFirestore.js
  1. Start the Backend Server
cd server
node index.js

The backend will run on: http://localhost:3001

  1. Start the Frontend App
cd client
npm run start

The frontend will be available at: http://localhost:3000 (now I just have journalPage and interviewPage.)

  1. Access the Application Open your browser and go to: http://localhost:3000/journal or http://localhost:3000/interview You should now see the journalPage and interviewPage running locally

  2. Run Backend Tests

cd server
npm test
